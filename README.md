# Симулятор Торговой Точки

WPF-приложение, моделирующее работу торговых точек с покупателями и службой доставки.

## Вариант 8

**Задание:** Торговая точка — смоделировать работу торговой точки.

## Структура решения

```
task-3.sln
├── TradingPointLib/          — библиотека классов (бизнес-логика)
```

## Классы и интерфейсы

| Класс/Интерфейс | Описание |
|---|---|
| `Product` | Товар с названием, ценой и количеством на складе |
| `Customer` | Покупатель с бюджетом, координатами и методом движения к цели |
| `TradingPoint` | Торговая точка с очередью покупателей, списком товаров и вероятностью покупки |
| `IDeliveryService` | Интерфейс службы доставки с методом `DeliverAsync` |
| `DeliveryService` | Асинхронная доставка товаров с задержкой (имитация времени доставки) |
| `ReflectionHelper` | Получение метаданных классов и динамическое создание объектов |

## События

- **ProductPurchased** — покупатель успешно купил товар
- **ProductOutOfStock** — товар закончился на складе, запускается доставка
- **DeliveryCompleted** — доставка прибыла, товар пополнен

## Реализованные требования

| Требование | Реализация |
|---|---|
| Делегаты/события | Пользовательский делегат `TradingEventHandler`, события `ProductPurchased`, `ProductOutOfStock`, `DeliveryCompleted` |
| Многопоточность | Асинхронный цикл симуляции (`Task`), `DeliverAsync` с `Task.Delay`, `Dispatcher.Invoke` для UI-потока |
| Рефлексия | `ReflectionHelper` — динамическое создание `DeliveryService` через `Activator.CreateInstance`, вывод метаданных классов |
| Динамическая визуализация | Покупатели перемещаются по Canvas к торговым точкам (20 FPS) |
| Несколько моделей | Возможность добавления нескольких торговых точек на форму |
| Вероятность событий | `BuyProbability` (75%) определяет вероятность покупки |
| MVVM | `BaseViewModel`, `RelayCommand`, `MainViewModel` с командами и `ObservableCollection` |
| Классы в отдельном модуле | Проект `TradingPointLib` |

## Как работает симуляция

1. Пользователь добавляет торговые точки (появляются на Canvas как зелёные прямоугольники)
2. Пользователь добавляет покупателей (появляются с краёв экрана как синие круги)
3. При запуске симуляции покупатели движутся к ближайшей торговой точке
4. По прибытии покупатель пытается купить товар (вероятность 75%)
5. Если товар закончился — автоматически вызывается служба доставки (3 сек)
6. Во время симуляции новые покупатели появляются автоматически
7. Все события отображаются в журнале справа
